  <action>
    If responsive issues were found in Task 1 checkpoint, apply fixes to css/style.css. Common fixes:

    1. **Image overflow (profile photo extends beyond viewport):**
       - Verify profile photo uses clamp() sizing: `width: clamp(120px, 30vw, 280px);`
       - If not using clamp, add: `max-width: 100%;` to ensure it scales with viewport
       - Verify max-width constraint matches or is narrower than parent container

    2. **Text overflow (content extends beyond 320px viewport):**
       - Check padding on containers (body, sections, cards)
       - Reduce padding at mobile breakpoint if needed: `@media (max-width: 640px) { .card { padding: 1rem; } }`
       - Ensure font-sizes use clamp() or scale appropriately: `font-size: clamp(0.875rem, 2.5vw, 1.125rem);`

    3. **Card layout not stacking on mobile:**
       - Verify card grid uses: `display: grid; grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));` (flexbox or grid)
       - Or add explicit @media rule: `@media (max-width: 768px) { .cards { display: grid; grid-template-columns: 1fr; } }`

    4. **Footer not stacking on mobile:**
       - Verify footer uses: `display: flex; flex-direction: row;` on desktop, `flex-direction: column;` on mobile
       - Or add @media rule: `@media (max-width: 640px) { .footer { flex-direction: column; } }`

    5. **Hamburger button not visible <768px:**
       - Verify CSS includes: `@media (min-width: 768px) { .nav__hamburger { display: none; } }`
       - Verify hamburger is `display: block;` by default

    6. **Touch targets <44x44px:**
       - Buttons, hamburger, tab buttons must be ≥44x44px (industry standard for touch)
       - Add padding if needed: `.btn { min-width: 44px; min-height: 44px; }`

    7. **Text unreadable at small viewport:**
       - Check line-height (should be 1.5-1.7 for body text)
       - Check font-size (should not be <14px for body text at 320px)
       - Add responsive sizing: `font-size: clamp(1rem, 2.5vw, 1.125rem);`

    **If no issues found:** No action needed. Proceed to Task 3.
  </action>
