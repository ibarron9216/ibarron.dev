<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 1: VoiceOver Accessibility Audit (Manual Testing)</name>
  <what-built>Comprehensive WCAG AA accessibility testing across all five pages using VoiceOver on macOS</what-built>

  <how-to-verify>
    **Test environment:** Safari on Mac with VoiceOver enabled (System Preferences > Accessibility > VoiceOver)

    **Testing checklist (apply to each page: index.html, research.html, teaching.html, students.html, contact.html):**

    1. **Navigation & Heading Hierarchy**
       - Open VoiceOver rotor (VO-U) and review Headings menu
       - Verify: One H1 per page, no skipped heading levels
       - Read page top to bottom with VO-Down Arrow, confirm announcements are clear

    2. **Hamburger Button (All Pages)**
       - Tab to hamburger button
       - Verify: Button is announced as "button", aria-label is descriptive (e.g., "Toggle navigation menu")
       - Press Space or Enter to toggle menu
       - Verify: aria-expanded changes from false → true → false as toggled
       - Verify: Menu is announced as "navigation" or list of links
       - Test on mobile viewport (<768px) to confirm hamburger visible and functional

    3. **Links (All Pages)**
       - Open VoiceOver rotor (VO-U) and review Links menu
       - Verify: All link text is descriptive (no "click here", "read more" alone; should be "Learn more about Research", etc.)
       - Verify: External links are announced or visually marked as opening in new tab (check for aria-label or title attribute)
       - Verify: mailto: links are announced as email addresses
       - Test keyboard Tab navigation through all links — verify logical left-to-right, top-to-bottom order

    4. **Tab Interface (Students Page Only)**
       - Tab to first tab button
       - Verify: Tab buttons are announced with role="tab", aria-selected value (true/false), and tab number (e.g., "Tab 1 of 3")
       - Press Right Arrow to move to next tab
       - Verify: aria-selected updates (false → true), content panel is announced and becomes active
       - Verify: Tab order is correct (left tab → middle tab → right tab)
       - Verify: Wrap-around works (right arrow from last tab goes to first tab) — check if phase 4 implemented this
       - Verify: Content panel is announced when switching tabs

    5. **Color Contrast & Text Readability**
       - Verify no yellow text appears on white backgrounds (WCAG AA violation if found)
       - Verify all nav/footer text on navy (#001E5F) is white or yellow (WCAG AA 4.5:1 or 3:1 contrast)
       - Verify all body text on white is navy (#001E5F) or dark gray (#707070) — readable at default zoom
       - Increase zoom to 200% and verify text remains readable and layouts don't break

    6. **Focus Indicators (Keyboard Testing)**
       - Disable mouse, use keyboard only (Tab, Shift+Tab, arrow keys, Space/Enter)
       - Tab through entire page — verify visible focus indicator on all focusable elements (buttons, links, tabs)
       - Verify focus indicator is not dark color on dark background or light on light (must have adequate contrast)
       - Verify focus indicator persists as element is interacted with

    7. **Image Alt Text (If Any)**
       - Open VoiceOver rotor (VO-U) and review Images menu
       - Verify: Profile photo placeholder (if CSS-based, no image tag needed) or any other images have alt text
       - Note: CSS-based profile box does not require alt text (decorative)

    8. **Dynamic Content & State Changes**
       - Toggle hamburger menu on/off — verify announcements reflect state change
       - Switch tabs on students page — verify announcements confirm which tab is now active
       - Click "Learn more" links — verify page load and navigation works

    **Documentation format for issues found:**
    Create structured list: Page | Element | Issue | WCAG Criterion | Severity (Critical/Major/Minor)
    Example:
    - research.html | Publication link "2023" | Link text not descriptive (just year) | WCAG 2.4.4 | Minor
    - students.html | Tab panel | aria-labelledby missing on panel | WCAG 1.3.1 | Major

    **Resume signal:** Type "approved" once all pages are tested, all issues are documented, and no blockers found. Or describe specific issues found (e.g., "Found 3 issues: focus indicator missing on tabs, yellow text on link in research section, heading level skipped").
  </how-to-verify>

  <resume-signal>Type "approved" when testing complete and documented, or provide list of issues to fix</resume-signal>
</task>

  <action>
    Based on issues documented in Task 1 checkpoint, apply fixes in priority order:

    **Critical fixes (accessibility blockers):**
    1. **Missing ARIA attributes on tab interface (if not already present):**
       - Verify each tab button has: role="tab", aria-selected="true|false", aria-controls="panel-id"
       - Verify each tab panel has: role="tabpanel", aria-labelledby="tab-id"
       - Verify tablist container has: role="tablist"
       - If missing, add to students.html

    2. **Missing or incorrect focus indicators:**
       - In css/style.css, ensure :focus state exists for all interactive elements: a, button, [role="tab"]
       - Focus style must be visible (e.g., outline: 3px solid var(--color-yellow); outline-offset: 2px;)
       - Verify focus is visible on navy background and white background (may need different outline colors)
       - Apply to: nav links, hamburger button, card links, tab buttons, external links

    3. **Yellow text on white background (if found):**
       - Search css/style.css for any rule that applies yellow (--color-yellow: #FFD82B) to text on white background
       - Change to navy (#001E5F) or dark gray (#707070)
       - Common place: body copy, labels, descriptions

    4. **Heading hierarchy issues (if found):**
       - Verify each page has exactly one H1
       - Verify no heading levels are skipped (e.g., H1 → H3 is a skip; must be H1 → H2)
       - Edit HTML heading tags to fix hierarchy

    5. **Image alt text (if any images exist):**
       - Add descriptive alt attribute to all <img> tags
       - For decorative images: alt="" (empty string)
       - For content images: alt="Clear, specific description"

    6. **Hamburger button semantic markup (verify/fix):**
       - Verify: <button class="nav__hamburger" aria-label="Toggle navigation menu" aria-expanded="false|true">
       - If not present, update to use button element with aria-label and aria-expanded

    7. **Link text clarity (if found in audit):**
       - Change generic "Learn more", "Read more", "Click here" to descriptive: "Learn more about Research", "View Teaching Page", etc.
       - For external links with target="_blank", consider adding aria-label (e.g., aria-label="ECE Department website (opens in new tab)")

    **Steps:**
    1. For each issue documented in Task 1:
       - Open the affected HTML file
       - Apply the fix described above
       - Save the file
    2. After all fixes applied:
       - Run automated validator: W3C HTML5 validation (no errors, warnings OK)
       - Prepare for re-testing in Task 3

    **Important:** Only fix issues documented in Task 1 checkpoint. Do not add new features or content. Do not change page layout or styling beyond focus indicators and color corrections.
