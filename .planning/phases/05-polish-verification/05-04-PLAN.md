    1. **Flexbox alignment differences (Safari vs Chrome):**
       - If items misaligned in Safari: add `display: flex; justify-content: center; align-items: center;`
       - Verify all flex containers have explicit alignment

    2. **Font rendering differences:**
       - If Inter font renders differently: ensure `font-family: 'Inter', sans-serif;` is applied to body
       - If Georgia serif renders differently: ensure `font-family: Georgia, serif;` for H1/H2

    3. **Tab borders or spacing issues:**
       - If tabs misaligned in Firefox: check CSS Grid gap values, adjust margins
       - Verify `box-sizing: border-box;` is applied globally (should be in css/style.css)

    4. **Focus state differences:**
       - Some browsers render outline differently; ensure `:focus { outline: 3px solid; outline-offset: 2px; }`
       - Test Safari's default focus (blue ring) vs Chrome's outline

    5. **Console errors in JavaScript:**
       - Check `initNav()`, `initFooterYear()`, `initTabs()` functions exist in js/main.js
       - Verify DOM selectors match HTML structure (e.g., `document.querySelector('.nav__hamburger')` exists in HTML)
       - Add error handling if missing: `if (hamburger) { hamburger.addEventListener('click', ...) }`

    **Common offline issues:**

    1. **Font not loading in file:// mode:**
       - Verify CSS includes: `@font-face { src: url('assets/inter-variable.woff2') format('woff2'); }`
       - Path must be relative, not absolute
       - Verify assets/inter-variable.woff2 file exists (not a symlink)

    2. **JavaScript errors in file:// mode:**
       - Avoid XMLHttpRequest or Fetch from file:// (use only if data is inline or local)
       - Verify js/main.js only modifies DOM, doesn't fetch external data
       - Test localStorage if used (should work in file:// if browser permits)

    3. **CORS or protocol errors:**
       - Ensure no `fetch()` or `axios()` calls to external APIs
       - If images needed, ensure they're local (not external CDN)
       - Verify all `src` and `href` attributes use relative paths (not absolute http://)

    **Steps:**
    1. For each issue documented in Tasks 1-2:
       - Open affected file (HTML, CSS, or JS)
       - Apply fix as described above
       - Save file
    2. Re-test in all three browsers (Safari, Chrome, Firefox) with same issue
    3. Verify fix resolves issue without introducing new ones

    **If no issues found:** No action needed. Proceed to Task 4.
