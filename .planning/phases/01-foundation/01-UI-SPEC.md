## Spacing Scale

Declared values (multiples of 4):

| Token | CSS Custom Property | Value | Usage |
|-------|---------------------|-------|-------|
| xs | `--spacing-xs` | 4px | Icon gaps, inline padding |
| sm | `--spacing-sm` | 8px | Compact element spacing, badge padding |
| md | `--spacing-md` | 16px | Default element spacing, card internal |
| lg | `--spacing-lg` | 24px | Card padding (TYP-08: 1.5rem = 24px), section internal |
| xl | `--spacing-xl` | 32px | Layout gaps |
| 2xl | `--spacing-2xl` | 48px | Major section breaks |

- Button vertical padding: 12px (0.75rem) — required by TYP-05
- Button horizontal padding: 28px (1.75rem) — required by TYP-05
- Touch targets: minimum 44px height on mobile — required by RESP-06

Source: REQUIREMENTS.md TYP-05, RESP-06. Scale defaults applied for tokens not explicitly stated in upstream.

## Typography

All `font-size` values use `clamp()` for fluid scaling where applicable. Declared values are the base (minimum) sizes.

**Exactly 4 size tokens. Exactly 2 weight values.**

| Role | CSS Custom Property | Size | Weight | Line Height | Font Family | Usage |
|------|---------------------|------|--------|-------------|-------------|-------|
| Heading / Display | `--font-size-heading` | `clamp(1.5rem, 3vw, 3.5rem)` | 700 (bold) | 1.2 | Georgia, serif | H1 and H2 headings |
| Subheading | `--font-size-subheading` | `clamp(1.125rem, 2vw, 1.5rem)` | 700 (bold) | 1.3 | Inter Variable, sans-serif | H3–H5, card titles, sub-sections |
| Body | `--font-size-body` | `clamp(1rem, 2vw, 1.125rem)` | 400 (regular) | 1.7 | Inter Variable, sans-serif | Body paragraphs, button labels, footer text |
| Label | `--font-size-label` | `0.875rem` (fixed) | 700 (bold) | 1.4 | Inter Variable, sans-serif | Section eyebrows, badge text, nav links |

**Weight assignments:**
- 700 (bold): H1, H2, H3, H4, H5, eyebrow labels, button labels, nav links, badge text
- 400 (regular): body paragraphs, footer text, card body copy

**Removed tokens (consolidated):**
- `--font-size-button` (was 1.125rem) — merged into `--font-size-body` (clamp 1rem–1.125rem); button labels use `--font-size-body` at weight 700
- `--font-size-eyebrow` (was 0.875rem standalone) — renamed to `--font-size-label`; same value, cleaner name covers all label/badge/eyebrow roles
- `--font-size-h1` and `--font-size-h2` (were separate) — merged into `--font-size-heading`; H1 uses the full clamp range, H2 is contextually the same token
- 600 (semibold) weight — removed; all previously semibold elements reassigned to 700 (bold)

Eyebrow styling (TYP-04): `text-transform: uppercase; letter-spacing: 0.1em;` applied via class, in addition to `--font-size-label` and weight 700.

Source: REQUIREMENTS.md TYP-01 through TYP-08, RESEARCH.md Phase Requirements table. Consolidated per checker revision 2026-03-25.

| Dominant (60%) | `--color-white` | #FFFFFF | Page background, card backgrounds, text on navy |
| Secondary (30%) | `--color-navy` | #001E5F | Nav bar, page headers, footer, H1 hero backgrounds |
| Accent (10%) | `--color-yellow` | #FFD82B | Reserved elements only (see below) |
| Hover/Interactive | `--color-meliora-blue` | #0045A0 | Primary button hover state only |
| Text body | `--color-dark-gray` | #707070 | Muted body text, eyebrow labels on white backgrounds |
| Card border | `--color-border` | #E8E8E8 | Card 1px border (TYP-08) |
| Butter background | `--color-butter` | #FFFBF0 | Research Overview and Teaching Philosophy section backgrounds |
| Destructive | — | — | No destructive actions in Phase 1 |

**Accent (#FFD82B) reserved for:**
1. Active nav link text and underline (`aria-current="page"` styled links)
2. Card top border — 4px yellow top border on highlight cards (TYP-08)
3. Profile photo placeholder "IB" initials text
4. Tab active state indicator (yellow text on navy background — ANIM-05)
5. Publication card left border (4px yellow left border — RES-10, Phase 3 preview pattern established here)

**Accent is NOT used for:** body text, general links, hover states on non-nav elements, or backgrounds other than the above.

**WCAG AA contrast verification (from RESEARCH.md FOUND-06):**
- Navy #001E5F on white #FFFFFF: 15.54:1 — PASS (min 4.5:1)
- White #FFFFFF on navy #001E5F: 15.54:1 — PASS
- Yellow #FFD82B on navy #001E5F: 11.18:1 — PASS
- Dark gray #707070 on white #FFFFFF: 4.95:1 — PASS
- Yellow #FFD82B on white: FAIL — yellow text MUST NOT appear on white backgrounds (A11Y-02)

Source: REQUIREMENTS.md FOUND-02, FOUND-06, TYP-06, TYP-07, TYP-08, A11Y-01, A11Y-02. CONTEXT.md brand decisions.

### Navigation

Structure: `<nav role="navigation">` with `<ul role="list">` links.

Note: RESEARCH.md flags that `role="menubar"` (specified in CONTEXT.md) is inappropriate per WAI-ARIA APG for site navigation. This spec adopts the disclosure nav pattern instead: `<nav>` with plain `<ul>` links and `aria-current="page"` on the active item. The hamburger button uses `aria-expanded` and `aria-controls`. This aligns with A11Y-04.

| Element | CSS class | ARIA attribute | Visual |
|---------|-----------|----------------|--------|
| Nav container | `.nav` | `role="navigation"` | White background, `position: sticky; top: 0`, box-shadow on scroll |
| Brand name | `.nav__brand` | — | Left-aligned, Georgia bold, navy text |
| Nav links | `.nav__links` | `role="list"` | Right-aligned, flex row, `--font-size-label` weight 700 |
| Active link | `.nav__link[aria-current]` | `aria-current="page"` | Yellow text (#FFD82B), yellow underline |
| Hamburger button | `.nav__hamburger` | `aria-expanded`, `aria-controls="nav-menu"`, `aria-label="Toggle navigation"` | 3-line icon, visible only at <768px |
| Mobile menu | `.nav__menu` | `id="nav-menu"` | Hidden by default, full-width dropdown |

## Interaction Contracts

### Hamburger Menu (JavaScript)

Trigger: click `.nav__hamburger`
Effect:
1. Toggle `aria-expanded` between `"false"` and `"true"` on the button
2. Toggle `.nav--open` class on `.nav` element
3. CSS `.nav--open .nav__menu { display: block }` shows mobile menu
4. CSS `.nav--open .nav__hamburger` renders 3 lines as × (X) pattern

Keyboard: button is `<button>` element — receives Enter/Space natively. No additional JS needed.

### Animations (CSS + `prefers-reduced-motion`)

All animations wrapped in:
```css
@media (prefers-reduced-motion: no-preference) { ... }
```

Page-load fade-in + slide-up:
- `@keyframes fadeInUp`: opacity 0→1, translateY(20px)→translateY(0)
- Duration: 0.5s, easing: `ease-out`
- Stagger: 0.1s delay per `.section:nth-child(n)` selector
- Applied to: all `<section>` and `<article>` elements at page load

Hover transitions (always active — not wrapped in reduced-motion, intentional: these are immediate responses, not gratuitous):
- Buttons: 0.2s ease on background-color, color, border-color
- Cards: 0.2s ease on box-shadow (subtle lift on hover)
- Nav links: 0.2s ease on color

### Focus Indicators

All interactive elements:
```css
:focus-visible {
  outline: 3px solid var(--color-yellow);
  outline-offset: 2px;
}
```
Exception: elements on navy backgrounds use white outline instead.

## CSS Architecture

### File: `css/style.css` (single file, ASSET-03)

Organization order (top to bottom):
1. `@font-face` declarations (Inter Variable)
2. `:root` custom properties — colors, spacing, fonts, breakpoints
3. CSS reset / base styles
4. Typography base styles (h1–h6, p, a)
5. Layout utilities (container, grid helpers)
6. Navigation component
7. Footer component
8. Button components
9. Card components
10. Tag pill / badge components
11. Profile photo placeholder
12. Animation keyframes + prefers-reduced-motion wrapper
13. Responsive breakpoint overrides (mobile → tablet → desktop)

### CSS Custom Properties in `:root`

```css
:root {
  /* Colors */
  --color-navy: #001E5F;
  --color-yellow: #FFD82B;
  --color-meliora-blue: #0045A0;
  --color-white: #FFFFFF;
  --color-dark-gray: #707070;
  --color-border: #E8E8E8;
  --color-butter: #FFFBF0;

  /* Spacing */
  --spacing-xs: 4px;
  --spacing-sm: 8px;
  --spacing-md: 16px;
  --spacing-lg: 24px;
  --spacing-xl: 32px;
  --spacing-2xl: 48px;
  --spacing-3xl: 64px;

  /* Typography — 4 size tokens, 2 weights (400 regular, 700 bold) */
  --font-sans: 'Inter Variable', system-ui, -apple-system, sans-serif;
  --font-serif: Georgia, 'Times New Roman', serif;
  --font-size-heading: clamp(1.5rem, 3vw, 3.5rem);
  --font-size-subheading: clamp(1.125rem, 2vw, 1.5rem);
  --font-size-body: clamp(1rem, 2vw, 1.125rem);
  --font-size-label: 0.875rem;

  /* Breakpoints (reference only — used in @media queries, not var()) */
  --breakpoint-mobile: 640px;
  --breakpoint-nav: 768px;
  --breakpoint-desktop: 1024px;
}
```

### File: `js/main.js` (single file, <200 lines)

Three discrete functions:
1. `initHamburger()` — nav toggle with aria-expanded
2. `initTabs()` — ARIA tab pattern with keyboard navigation (Phase 4 use)
3. `initFooterYear()` — `document.querySelector('.footer__year').textContent = new Date().getFullYear()`

## Responsive Behavior Contract

| Breakpoint | Layout |
|------------|--------|
| < 640px (mobile) | Single column, hamburger nav, stacked footer |
| 640px–1023px (tablet) | Two-column where applicable, hamburger nav still active |
| < 768px | Nav collapses to hamburger (NAV-02) |
| ≥ 768px | Nav links shown inline, hamburger hidden |
| ≥ 1024px (desktop) | Full multi-column layouts active |

All multi-column layouts (hero, footer, contact, card grids) stack to single column below their respective breakpoints using CSS Grid `grid-template-columns` overrides.

Minimum supported width: 320px (RESP-01).
