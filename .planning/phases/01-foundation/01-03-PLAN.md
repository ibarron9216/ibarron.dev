  <action>
    Perform HTML5 structure and responsive design validation:

    **1. HTML5 Semantic Validation:**
    - Verify using W3C HTML Checker (https://validator.w3.org/) or Nu HTML Checker tool
    - Check for:
      - One `<html>` element with `lang="en"` attribute
      - One `<head>` with `<title>` and meta tags (charset, viewport)
      - One `<body>` with semantic landmarks: `<header>`, `<nav>`, `<main>`, `<footer>`
      - One `<h1>` per page (required exactly once)
      - No skipped heading levels
      - Proper nesting of list elements (`<ul>` contains `<li>`)

    **2. Responsive Design Breakpoints — Test at Three Viewport Sizes:**

    Using browser DevTools (Chrome, Firefox, Safari):

    **Mobile (320px width):**
    - Hamburger button VISIBLE
    - Nav links HIDDEN (until hamburger clicked)
    - Main content single column
    - Footer stacked single column
    - All text readable (no horizontal scroll needed)

    **Tablet (768px width):**
    - Hamburger button HIDDEN
    - Nav links VISIBLE inline
    - Main content single or two-column depending on layout
    - Footer appropriate for tablet size
    - All spacing and typography scale correctly

    **Desktop (1024px+ width):**
    - Full nav visible with all links
    - Full desktop layouts active
    - Multi-column layouts where applicable
    - All elements properly spaced per design system

    **3. Responsive Media Query Verification:**
    Confirm css/style.css contains:
    - `@media (max-width: 639px)` rules for mobile single-column
    - `@media (min-width: 640px)` rules for tablet two-column
    - `@media (min-width: 768px)` rules for nav collapse (hamburger hidden, menu shown)
    - `@media (min-width: 1024px)` rules for desktop full layouts

    **4. Documentation:**
    Create a test report noting:
    - W3C validation result (pass/fail, any errors/warnings)
    - Responsive test results at 320px, 768px, 1024px
    - Any issues found and how they were fixed
  </action>
