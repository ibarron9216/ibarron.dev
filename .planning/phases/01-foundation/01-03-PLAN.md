  <action>
    Perform HTML5 structure and responsive design validation:

    **1. HTML5 Semantic Validation:**
    - Check for W3C compliance using the Nu HTML Checker (https://validator.w3.org/) or command-line tool
    - Verify:
      - One `<html>` element with `lang="en"` attribute
      - One `<head>` with `<title>` and meta tags (charset, viewport)
      - One `<body>` with semantic landmarks: `<header>`, `<nav>`, `<main>`, `<footer>`
      - One `<h1>` per page (heading hierarchy starts at H1, no H2 before H1, no skipped levels)
      - All `<button>` elements have accessible text (aria-label or text content)
      - No empty `<a>` tags without aria-label
      - Proper nesting of list elements (`<ul>` contains `<li>`)
      - Form elements (if any) properly labeled

    **2. Responsive Design Breakpoints (Manual or Tool-Based Testing):**

    Test at three viewport sizes using browser DevTools (Chrome, Firefox, Safari):

    **Mobile (320px width):**
    - Hamburger button VISIBLE
    - Nav links HIDDEN (until hamburger clicked)
    - Main content single column
    - Footer stacked single column
    - All text readable (no horizontal scroll needed)
    - Touch targets (buttons, links) at least 44px tall per RESP-06

    **Tablet (768px width):**
    - Hamburger button HIDDEN (or transitioning)
    - Nav links VISIBLE inline (no dropdown)
    - Main content still single column or two-column depending on page layout
    - Footer two-column if applicable
    - All spacing and typography scale correctly with clamp()

    **Desktop (1024px+ width):**
    - Full multi-column layouts active (hero two-column if present)
    - Nav fully horizontal with all links visible
    - Footer two-column layout
    - All elements properly spaced per design system

    **3. Responsive Media Query Verification:**
    In css/style.css, confirm:
    - `@media (max-width: 639px)` rules exist for mobile single-column
    - `@media (min-width: 640px)` rules exist for tablet two-column
    - `@media (min-width: 768px)` rules exist for nav collapse (hamburger hidden, menu shown)
    - `@media (min-width: 1024px)` rules exist for desktop full layouts
    - `clamp()` is used for font-size, not fixed breakpoint font sizes (allows smooth scaling between breakpoints)

    **4. Focus Order and Interactive Element Validation:**
    Use keyboard navigation (Tab key):
    - Starting from top of page, tab through all interactive elements
    - Order should follow visual left-to-right, top-to-bottom flow
    - Hamburger button receives focus before nav links (on mobile)
    - Nav links receive focus in visual order (Home, Research, Teaching, Students, Contact)
    - Footer links (if any) receive focus last
    - No focus trap or keyboard navigation loops
    - All elements with focus receive visible indicator (outline, highlight, underline per :focus-visible)

    **5. Heading Hierarchy Validation:**
    In index.html (and note for later pages):
    - One `<h1>` element (required, exactly one per page per A11Y-08)
    - No `<h2>` or deeper without `<h1>` first
    - No skipped heading levels (e.g., H1 → H3, skip H2)
    - Heading text is descriptive and meaningful (not "Content" or "Title")

    **6. Color Contrast Verification (spot-check):**
    Using browser DevTools Accessibility Inspector or WebAIM Contrast Checker (https://webaim.org/resources/contrastchecker/):
    - Navy (#001E5F) on white background: pass WCAG AA (15.54:1 per UI-SPEC)
    - White on navy: pass WCAG AA
    - Yellow (#FFD82B) on navy: pass WCAG AA
    - Dark gray (#707070) on white: pass WCAG AA
    - Yellow on white: MUST FAIL per A11Y-02 (this is intentional — yellow is NEVER on white)

    **7. Screen Reader Testing (Optional but Recommended):**
    Using browser extension (NVDA for Windows, VoiceOver on macOS):
    - Nav announced as "navigation" landmark
    - Hamburger button announced as "Toggle navigation" (aria-label)
    - Current page link announced as "current page" (aria-current)
    - Headings announced with level ("Heading 1", "Heading 3")
    - No unnecessary aria-labels on elements that have text content
    - Footer announced as "contentinfo" landmark

    **Documentation:**
    Create a test report noting:
    - W3C validation result (pass/fail, any errors/warnings)
    - Responsive test results at 320px, 768px, 1024px
    - Focus order sequence (tab order)
    - Heading hierarchy (H1 count, levels)
    - Color contrast spot-checks
    - Any issues found and how they were fixed
