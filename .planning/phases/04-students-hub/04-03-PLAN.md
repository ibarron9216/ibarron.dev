<how-to-verify>

## Visual & Interaction Checklist

Open `students.html` in a web browser (Chrome, Firefox, or Safari) and test the following:

### 1. Page Header (Visual)
- [ ] Navy background section with minimum 35vh height visible
- [ ] Large white H1 heading reads "Students Hub"
- [ ] Yellow eyebrow text above H1 reads "Resources for current, prior, and prospective students"
- [ ] Header spacing and padding feel balanced, consistent with Research/Teaching/Contact pages
- [ ] Header is centered and not overlapping navigation bar

### 2. Tab Buttons (Visual & Interaction)
- [ ] Three tab buttons are visible in a horizontal row
- [ ] Tab buttons are styled as navy pills with white text (active "Current Students")
- [ ] Inactive tabs ("Prior Students", "Prospective Students") are transparent background with navy text
- [ ] Hover over an inactive tab button: background or text color changes subtly (Meliora Blue)
- [ ] Tab buttons have visible focus indicator (outline or highlight) when you press Tab to focus them
- [ ] Tab buttons are at least 44px tall (comfortable to click/tap)
- [ ] Gap between tabs is consistent (approximately 16px)
- [ ] On mobile (320px width): Tab buttons fit on one line without horizontal scroll, or wrap gracefully if needed

### 3. Tab Panels (Content & Styling)
- [ ] "Current Students" panel is visible with white background
  - [ ] Contains office hours information or placeholder
  - [ ] Contains "Access course materials on Blackboard" link
  - [ ] Contains "Email me directly at ibarron@ur.rochester.edu" link
  - [ ] Contains accommodations or accessibility information
  - [ ] All text uses first-person voice (I, my, me)
- [ ] "Prior Students" panel has a `.card--highlighted` with yellow top border
  - [ ] Card title reads "Recommendation Letters"
  - [ ] Card contains bulleted list of requirements (4-5 items)
  - [ ] First-person language in bulleted items
- [ ] "Prospective Students" panel has a `.card--highlighted` with yellow top border
  - [ ] Card title reads "Undergraduate Research Opportunities"
  - [ ] Card contains bulleted list of application requirements (4-5 items)
  - [ ] Card has a call-to-action mentioning email address
  - [ ] First-person language in bulleted items

### 4. Tab Switching — Click (Interaction)
- [ ] Click on "Prior Students" tab
  - [ ] "Current Students" panel fades out smoothly (0.2s fade)
  - [ ] "Prior Students" panel fades in smoothly
  - [ ] "Prior Students" button shows navy background + white text
  - [ ] "Current Students" and "Prospective Students" buttons show transparent background + navy text
- [ ] Click on "Prospective Students" tab
  - [ ] "Prior Students" panel fades out
  - [ ] "Prospective Students" panel fades in
  - [ ] "Prospective Students" button shows navy background + white text
- [ ] Click on "Current Students" tab again
  - [ ] Fade transition is smooth and seamless
  - [ ] "Current Students" button shows active styling

### 5. Tab Switching — Arrow Keys (Keyboard Interaction)
- [ ] Click on any tab button to focus it
- [ ] Press Right arrow key
  - [ ] Focus moves to next tab button to the right
  - [ ] Panel switches to match the newly focused tab
  - [ ] Fade transition is smooth
- [ ] Keep pressing Right arrow until you reach "Prospective Students" tab, then press Right once more
  - [ ] Focus wraps to "Current Students" tab (circular navigation)
  - [ ] Panel switches to Current Students
- [ ] Press Left arrow key
  - [ ] Focus moves to previous tab button
  - [ ] Panel switches to match
- [ ] From "Current Students" tab, press Left arrow
  - [ ] Focus wraps to "Prospective Students" tab (circular wrapping in reverse)
  - [ ] Panel switches to Prospective Students

### 6. Tab Switching — Tab Key (Keyboard Navigation)
- [ ] Start at top of page (focus on hamburger button on mobile, or first nav link on desktop)
- [ ] Press Tab key repeatedly to navigate forward through the page
  - [ ] Focus moves through nav links, then lands on first tab button ("Current Students")
  - [ ] Continue pressing Tab to move through remaining tab buttons ("Prior Students", "Prospective Students")
  - [ ] After last tab button, press Tab once more: focus moves into active panel (likely the email link or first link in the panel)
  - [ ] Verify panel content is interactive (links are clickable, text is readable)
- [ ] Verify Shift+Tab reverses the navigation order
  - [ ] Start at bottom of active panel content (last link or button)
  - [ ] Press Shift+Tab to move backward
  - [ ] Focus moves through tab buttons in reverse order
  - [ ] Exits to header/nav area as expected

### 7. Links & External Interactions
- [ ] Click on "Access course materials on Blackboard" link
  - [ ] Link is blue, underlined, or styled as a link
  - [ ] Clicking opens Blackboard (or shows link is present if Blackboard link is placeholder)
  - [ ] If placeholder, URL is present (e.g., href="https://blackboard.com")
- [ ] Click on "ibarron@ur.rochester.edu" email link
  - [ ] Opens default email client (or shows composed email preview in browser)
  - [ ] Email address is correct: ibarron@ur.rochester.edu
- [ ] Look for any "Graduate Admissions" or similar external link in Prospective Students panel
  - [ ] Link opens in a new tab (target="_blank")
  - [ ] Link text is descriptive and clear

### 8. Navigation Bar Consistency
- [ ] Hamburger button on mobile (at 320px width)
  - [ ] Hamburger is visible and functional
  - [ ] Click hamburger to open mobile nav
  - [ ] Verify "Students" link shows active state (yellow text or underline)
  - [ ] Other nav links ("Home", "Research", "Teaching", "Contact") are present but not active
- [ ] On desktop (1024px+ width)
  - [ ] Nav bar shows all five links horizontally
  - [ ] "Students" link is active (yellow text or underline)

### 9. Footer Consistency
- [ ] Footer displays at bottom of page with navy background and white text
- [ ] Year in footer updates dynamically (e.g., "© 2026" or similar)
- [ ] Footer links are styled consistently (white text, underline on hover)

### 10. Responsive Testing
Test at three screen widths (use browser DevTools device emulation):

**Mobile (320px width):**
- [ ] Page layout is single-column, readable without horizontal scroll
- [ ] Tab buttons fit on one line or wrap gracefully
- [ ] Panel content is fully readable (font size appropriate, padding sufficient)
- [ ] Links and buttons are at least 44px tall for touch targets
- [ ] No overlapping elements, no text cutoff

**Tablet (768px width):**
- [ ] Tab buttons and panel content display with appropriate padding
- [ ] Layout is readable and not cramped
- [ ] Cards (Recommendation Letters, Undergraduate Research) have good spacing

**Desktop (1024px+ width):**
- [ ] Tab interface is centered within max-width 1100px container
- [ ] Panel content has adequate padding and line length is readable
- [ ] Card styling (yellow top border) is clear and visible

### 11. Accessibility Testing
- [ ] Open DevTools → Inspector
- [ ] Click on a tab button, check its computed styles:
  - [ ] Verify aria-selected="true" (or "false" for inactive)
  - [ ] Verify tabindex="0" for active tab, tabindex="-1" for inactive
- [ ] Check Accessibility Inspector:
  - [ ] role="tablist" is present on tab container
  - [ ] role="tab" is present on each button
  - [ ] role="tabpanel" is present on each panel
  - [ ] No ARIA errors or warnings
- [ ] Check heading hierarchy:
  - [ ] One H1 per page (in page header)
  - [ ] No skipped heading levels (e.g., H1 → H3 would skip H2)
  - [ ] Card titles inside panels are H3 or H4 (appropriate sub-level)

### 12. prefers-reduced-motion Testing
- [ ] Open DevTools → Rendering tab
- [ ] Check "Emulate CSS media query (prefers-reduced-motion: reduce)"
- [ ] Click on a different tab
  - [ ] Panel switches instantly (no fade animation)
  - [ ] Content is still visible and readable
  - [ ] Interaction is still functional
- [ ] Uncheck the emulation
  - [ ] Panel fade animation is smooth again

### 13. Browser Console
- [ ] Open DevTools → Console tab
- [ ] Interact with tabs (click, arrow keys, tab key)
- [ ] Verify no red error messages appear
- [ ] No warnings related to students.html, initTabs(), or ARIA attributes

### 14. Overall Visual Consistency
- [ ] Students Hub page design matches Research, Teaching, and Contact pages
  - [ ] Navy header style consistent
  - [ ] Typography (heading sizes, weights) consistent
  - [ ] Card styling (white bg, border, yellow top border) consistent
  - [ ] Spacing and padding feel aligned with other pages
  - [ ] Button styling (pill-shaped, colors) feels integrated
- [ ] No visual glitches, overlapping text, or misaligned elements
- [ ] Page feels polished and complete
