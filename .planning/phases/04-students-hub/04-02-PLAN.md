## Keyboard Navigation Pattern (W3C APG standard)

Tab key behavior:
- Focus moves through visible tab buttons left to right
- After last tab, Tab moves to first focusable element inside active panel
- Shift+Tab reverses flow

Arrow key behavior (already in existing code):
- Left arrow: Move to previous tab (wrap to last if at first)
- Right arrow: Move to next tab (wrap to first if at last)
- Focus must move to the new active tab

Implementation:
- Update tabindex on all tabs: active tab = tabindex="0", inactive = tabindex="-1"
- Update tabindex when active tab changes
- Move focus to active tab with .focus()

## ARIA Attributes Reference

Tab interface requires:
- `<div role="tablist">` — Container for all tab buttons
- `<button role="tab" aria-selected="true|false" aria-controls="panel-id">` — Tab button
  - aria-selected: "true" for active, "false" for inactive
  - aria-controls: ID of the panel this tab controls
  - tabindex: "0" for active, "-1" for inactive
- `<div role="tabpanel" id="panel-id">` — Panel container
  - id: Matches aria-controls on controlling tab
  - hidden: Attribute present if panel is not visible (managed by JS)

Optional but helpful:
- `aria-labelledby="tab-id"` on tabpanel (points to the controlling tab)

**Step 2: Helper function to switch tabs**
Create an internal switchTab(targetIndex) function to avoid code duplication. This function:
- Finds the target tab and panel by index
- Hides the current panel (fade out)
- Deselects current tab
- Shows the new panel (fade in)
- Selects new tab
- Updates tabindex on all tabs (new active = 0, others = -1)
- Moves focus to new tab

```javascript
function switchTab(targetIndex) {
  const currentTab = tablist.querySelector('[aria-selected="true"]');
  const currentPanel = document.getElementById(currentTab.getAttribute('aria-controls'));
  const targetTab = tabs[targetIndex];
  const targetPanel = document.getElementById(targetTab.getAttribute('aria-controls'));

  // Hide current panel with fade
  fadeOutPanel(currentPanel, () => {
    currentPanel.hidden = true;
  });

  // Show new panel with fade
  targetPanel.hidden = false;
  // Force reflow to ensure CSS transition applies
  void targetPanel.offsetHeight;
  fadeInPanel(targetPanel);

  // Update ARIA attributes
  currentTab.setAttribute('aria-selected', 'false');
  targetTab.setAttribute('aria-selected', 'true');

  // Update tabindex
  tabs.forEach((tab, idx) => {
    tab.setAttribute('tabindex', idx === targetIndex ? '0' : '-1');
  });

  // Move focus to new tab
  targetTab.focus();
}
```

**Step 3: Fade out helper function**
Creates the opacity fade-out effect. Respects prefers-reduced-motion:

```javascript
function fadeOutPanel(panel, onComplete) {
  const prefersReducedMotion = window.matchMedia('(prefers-reduced-motion: reduce)').matches;

  if (prefersReducedMotion) {
    // Instant: no animation
    onComplete();
    return;
  }

  // Fade out: set opacity to 0, wait for transition to complete
  panel.style.opacity = '0';
  const handleTransitionEnd = () => {
    panel.removeEventListener('transitionend', handleTransitionEnd);
    onComplete();
  };
  panel.addEventListener('transitionend', handleTransitionEnd);

  // Fallback timeout (150ms extra buffer for slow transitions)
  const timeoutId = setTimeout(() => {
    panel.removeEventListener('transitionend', handleTransitionEnd);
    onComplete();
  }, 300); // 0.2s transition + 100ms buffer
}
```

**Step 4: Fade in helper function**
Creates the opacity fade-in effect. Respects prefers-reduced-motion:

```javascript
function fadeInPanel(panel) {
  const prefersReducedMotion = window.matchMedia('(prefers-reduced-motion: reduce)').matches;

  if (prefersReducedMotion) {
    // Instant: just set opacity to 1
    panel.style.opacity = '1';
    return;
  }

  // Fade in: set opacity to 1 (CSS transition applies automatically)
  panel.style.opacity = '1';
}
```

**CRITICAL IMPLEMENTATION NOTES:**

1. **Fade animation relies on CSS transitions:**
   - CSS must define `transition: opacity 0.2s ease` on `[role="tabpanel"]`
   - If prefers-reduced-motion: no-preference matches, the CSS transition applies
   - If prefers-reduced-motion: reduce matches, browser does not apply the transition (instant)
   - JavaScript detects prefers-reduced-motion with `window.matchMedia('(prefers-reduced-motion: reduce)')`

2. **Hidden attribute semantics:**
   - Use `hidden` attribute (not `display: none` in CSS) to hide panels
   - `[role="tabpanel"][hidden]` should have `display: none` in CSS (or visibility: hidden)
   - Ensures screen readers don't announce hidden panels

3. **Focus management:**
   - After arrow key navigation, call `targetTab.focus()` to move keyboard focus
   - Tab key will then move through remaining tabs and into the active panel
   - Shift+Tab reverses the flow

4. **Reflow trigger:**
   - Use `void element.offsetHeight` to force reflow before applying fade-in
   - Ensures browser applies the 0 opacity state before transitioning to 1
   - Without this, the fade-in animation may not be visible

5. **Transition end event:**
   - Use transitionend event to know when fade-out completes before setting hidden=true
   - Add timeout fallback (300ms) in case transitionend doesn't fire (browser inconsistencies)
