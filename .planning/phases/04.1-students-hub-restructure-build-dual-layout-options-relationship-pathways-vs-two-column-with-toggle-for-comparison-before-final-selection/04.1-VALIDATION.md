## Manual-Only Verifications

| Behavior | Requirement | Why Manual | Test Instructions |
|----------|-------------|------------|-------------------|
| Layout 1 sidebar displays on desktop | CONTEXT: Two-Column design | Visual layout inspection | Open students.html on 1024px+ viewport, verify left sidebar with 4 topic links visible |
| Layout 1 converts to select menu on mobile | CONTEXT: Mobile breakpoint at 768px | Responsive design testing | Resize to 640px and 768px, verify sidebar becomes a `<select>` dropdown |
| Layout 2 cards stack vertically | CONTEXT: Expandable Cards design | Visual layout inspection | Open students.html on all viewport sizes, verify cards stack in single column |
| Cards expand/collapse smoothly | CONTEXT: Animation (0.2s transition) | Animation playback | Click cards on Layout 2, verify smooth expand/collapse animation |
| Toggle switch changes both layouts | CONTEXT: Comparison mechanism | Interaction testing | Click layout toggle at top of page, verify both Layout 1 AND Layout 2 render correctly on next paint |
| localStorage persists selection | CONTEXT: Session persistence | State management testing | Select Layout 1, reload page, verify Layout 1 still selected. Repeat for Layout 2. Check browser DevTools > Application > localStorage for `'ibarron:studentsLayout'` |
| All 4 topics display in each layout | CONTEXT: Core Topics (Get Started, Collaboration, Professional Recommendations, Student Impact/Outcomes) | Content verification | Switch between layouts, verify all 4 topics appear in correct order and with correct content in both layouts |
| First-person voice consistent | CLAUDE.md requirement | Content review | Scan visible content for "I/my/me" pronouns, verify no third-person phrases like "the faculty" or "the professor" |
| ARIA labels on interactive elements | WCAG 2.1 AA compliance | Accessibility inspection | DevTools Inspector: all `<button>`, `<select>`, `role="switch"`, etc. have `aria-label` or descriptive text |
| Tab focus visible on all elements | WCAG 2.1 AA compliance | Keyboard navigation | Tab through entire page on both layouts, verify focus outline visible on every interactive element |
| Sidebar/select navigation reachable by keyboard | WCAG 2.1 AA compliance | Keyboard navigation (Layout 1) | Tab to sidebar/select, use arrow keys or click to select topic, verify keyboard alone can navigate all topics |
| Card expand buttons reachable by keyboard | WCAG 2.1 AA compliance | Keyboard navigation (Layout 2) | Tab to each card's expand button, press Enter/Space, verify card expands/collapses |
