## Objective

Simplify the Students Hub to use sidebar navigation exclusively. Remove the accordion cards layout, toggle button, and all related CSS and JavaScript for dual-layout switching.

## Tasks

### Task 1: Update students.html to sidebar-only layout

Remove:
- The layout toggle button (`.layout-toggle`)
- Entire Layout 2 section (accordion cards with `role="region"`, `aria-expanded`)
- `data-layout="2"` attribute
- Mobile select dropdown (tied to Layout 1 selection)

Keep:
- Page header
- Sidebar navigation (4 topic buttons with `aria-selected`)
- 4 content panels displayed on sidebar selection
- First-person voice and all content

Make sidebar the only visible layout, no toggle needed.

**Files modified:** `students.html`

### Task 2: Simplify CSS — remove accordion and toggle styles

Remove from `css/style.css` (Section 19):
- Section 19a: `[data-layout]` attribute selectors (no longer needed, single layout)
- Section 19b: Layout toggle switch styles (pill button, aria-checked states)
- Section 19e: Expandable accordion cards max-height animation
- Keep Section 19c: Layout 1 two-column sidebar (now THE layout)
- Keep Section 19d: Mobile select dropdown? OR remove if not used

Update grid/spacing to work as single layout without layout switching logic.

**Files modified:** `css/style.css`

### Task 3: Simplify JavaScript — remove toggle and cards functions

Remove from `js/main.js`:
- `initLayoutToggle()` — no longer needed
- `initExpandableCards()` — accordion not in use
- Remove `ibarron:studentsLayout` localStorage references

Keep:
- `initSidebarNav()` — sidebar button selection and content switching

Update `DOMContentLoaded` to call only `initSidebarNav()` (and `initTabs()` for other pages).

**Files modified:** `js/main.js`

### Task 4: Verify sidebar-only layout

Quick check:
- [ ] students.html loads showing only sidebar + panels, no cards, no toggle

- [ ] Mobile (320px): sidebar layout adapts (single column if needed)
